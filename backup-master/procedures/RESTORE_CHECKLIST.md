# Checklist de restauração

- [ ] Arquivo possui data e origem.
- [ ] Tamanho foi registrado.
- [ ] SHA-256 foi conferido.
- [ ] Histórico Git abre e passa em `git fsck --full`.
- [ ] Banco passa no teste de integridade.
- [ ] Schema, RLS, funções e Storage estão documentados.
- [ ] Aplicação compila em ambiente limpo.
- [ ] Variáveis necessárias estão listadas sem valores secretos.
- [ ] URL, commit e versão foram registrados.
- [ ] Resultado do teste possui data e responsável.
